# Example Idea Note

## Summary

This is a reference idea note showing how to capture a possible future vault or product improvement.

## Description

This example proposes adding a lightweight “decision log” note type later so important business and product decisions can be tracked separately from meeting notes and project updates.

## Why it matters

As the vault grows, decision-making context can become harder to trace. A dedicated decision note pattern could make strategic memory cleaner.

## Opportunity

- Decisions become easier to search and review
- Product and operations history becomes more explicit
- Important choices can be linked back into [[MEMORY]] and [[ScriptLab8_I3]]

## Risks or concerns

- Too many note types can make the vault harder to use
- A decision log only works if it is actually maintained
